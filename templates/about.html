  • income (numeric): Annual income of the applicant (in decile form). Ranges between [0.1, 0.9].
  • name_email_similarity (numeric): Metric of similarity between email and applicant's name. Higher values represent higher similarity. Ranges between [0, 1].
  • prev_address_months_count (numeric): Number of months in the previous registered address of the applicant, i.e., the applicant's previous residence, if applicable. Ranges between [-1, 380] months (-1 is a missing value).
  • current_address_months_count (numeric): Months in the currently registered address of the applicant. Ranges between [-1, 429] months (-1 is a missing value).
  • customer_age (numeric): Applicant's age in years, rounded to the decade. Ranges between [10, 90] years.
  • days_since_request (numeric): Number of days passed since the application was done. Ranges between [0, 79] days.
  • intended_balcon_amount (numeric): Initial transferred amount for the application. Ranges between [-16, 114] (negatives are missing values).
  • zip_count_4w (numeric): Number of applications within the same zip code in the last 4 weeks. Ranges between [1, 6830].
  • velocity_6h (numeric): Velocity of total applications made in the last 6 hours, i.e., the average number of applications per hour in the last 6 hours. Ranges between [-175, 16818].
  • velocity_24h (numeric): Velocity of total applications made in the last 24 hours, i.e., the average number of applications per hour in the last 24 hours. Ranges between [1297, 9586].
  • velocity_4w (numeric): Velocity of total applications made in the last 4 weeks, i.e., the average number of applications per hour in the last 4 weeks. Ranges between [2825, 7020].
  • bank_branch_count_8w (numeric): Number of total applications in the selected bank branch in the last 8 weeks. Ranges between [0, 2404].
  • date_of_birth_distinct_emails_4w (numeric): Number of emails for applicants with the same date of birth in the last 4 weeks. Ranges between [0, 39].
  • credit_risk_score (numeric): Internal score of application risk. Ranges between [-191, 389].
  • email_is_free (binary): Domain of the application email (either free or paid).
  • phone_home_valid (binary): Validity of the provided home phone.
  • phone_mobile_valid (binary): Validity of the provided mobile phone.
  • bank_months_count (numeric): How old is the previous account (if held) in months. Ranges between [-1, 32] months (-1 is a missing value).
  • has_other_cards (binary): If the applicant has other cards from the same banking company.
  • proposed_credit_limit (numeric): Applicant's proposed credit limit. Ranges between [200, 2000].
  • foreign_request (binary): If the origin country of the request is different from the bank's country.
  • session_length_in_minutes (numeric): Length of user session in the banking website in minutes. Ranges between [-1, 107] minutes (-1 is a missing value).
  • keep_alive_session (binary): User option on session logout.
  • device_distinct_emails (numeric): Number of distinct emails in the banking website from the used device in the last 8 weeks. Ranges between [-1, 2] emails (-1 is a missing value).
  • month (numeric): Month when the application was made. Ranges between [0, 7].
  • payment_type (categorical): Credit payment plan type. 5 possible (anonymized) values.
  • employment_status (categorical): Employment status of the applicant. 7 possible (anonymized) values.
  • housing_status (categorical): Current residential status for the applicant. 7 possible (anonymized) values.
  • source (categorical): Online source of the application. Either browser (INTERNET) or app (TELEAPP).
  • device_os (categorical): Operating system of the device that made the request. Possible values are: Windows, macOS, Linux, X11, or other.
